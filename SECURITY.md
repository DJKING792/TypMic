# Security Policy

## Supported Versions

PhoneMic is a single-branch hobby/utility project. Security fixes land on `main` only.

| Version | Supported |
| ------- | --------- |
| main    | ✅        |

## Reporting a Vulnerability

If you discover a security issue, **please do not open a public issue.** Instead, report it privately:

1. Use GitHub's [private vulnerability reporting](https://github.com/DJKING792/PhoneMic/security/advisories/new) on this repo, **or**
2. Email the maintainer (see profile) with the details.

We aim to acknowledge within a few days and will coordinate a fix and disclosure timeline with you.

## Security Model (read this before deploying)

PhoneMic is designed for a **trusted local network only**:

- The server listens on `0.0.0.0:8443` with **no authentication by default**.
- All traffic is served over a **locally self-signed HTTPS** certificate (generated by the server on first run). The API key is **never** sent to the phone; the phone only streams audio.
- Your `MIMO_API_KEY` lives only in a local `.env` file (blocked by `.gitignore`). It is sent **only** to Xiaomi's MiMo ASR endpoint, never to any other party.
- The self-signed certificate embeds the PC's LAN IP in its SAN and is valid for ≤ 398 days, satisfying Apple/iOS requirements.

### Hardening checklist

- Run it only on a network you trust (your home/office Wi-Fi). **Do not expose port 8443 to the public internet.**
- If you must reach it remotely, put it behind a VPN or a reverse proxy with real TLS + authentication.
- On iPhone, after installing `rootCA.pem` you must also enable **Full Trust** at `Settings → General → About → Certificate Trust Settings`.
- Rotate/delete your `MIMO_API_KEY` if a device you don't trust was ever on the same network while the server was running.
